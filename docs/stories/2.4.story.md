# Story 2.4: Agenda — Visualização (FullCalendar)

## Status

Draft

## Story

**Como** owner/admin/professional,
**Eu quero** ver a agenda de toda a clínica ou filtrada por profissional, em visualizações dia/semana/mês,
**Para que** eu tenha contexto visual antes de marcar novos agendamentos.

## Complexidade

**T-shirt:** L (~10 pontos · 2 dias) — Story não-trivial por integrar biblioteca externa com Server Components + drag-and-drop.

## Contexto

ADR-002 escolhe FullCalendar (`@fullcalendar/react` + plugins `daygrid`, `timegrid`, `resource-timegrid`, `interaction`). Schema `appointments` já tem EXCLUDE constraint contra double-book. Esta story entrega apenas **visualização**; agendamento (criar/editar) é a Story 2.5.

## Acceptance Criteria

### AC1 — Render

1. `/agenda` protegida por viewer+; desktop mostra visualização semana por default, mobile mostra dia.
2. Toggle dia/semana/mês no topo.
3. Navegação: hoje, anterior, próximo.
4. Dado `professional_id` via query param, filtra; senão mostra todos.

### AC2 — Recursos

1. View "resource-timegrid" (semana com coluna por profissional) quando filtro "todos".
2. Cor do evento vem de `appointments.color` ou fallback para `service_category.color`.

### AC3 — Eventos

1. Cada evento exibe: nome do paciente, serviço, horário.
2. Clicar abre sheet com detalhes e botões (editar — Story 2.5+; concluir — Story 2.6; cancelar — Story 2.6).
3. Status "cancelled" aparece com opacity 0.4.

### AC4 — Performance

1. Query server filtra por `starts_at BETWEEN range` + `org_id` (RLS cuida do resto).
2. Carrega ≤ 200 eventos por view. Paginação não aplicável (time-bound).

### AC5 — Mobile

1. Layout timegrid funcional no mobile (≥ 375px).
2. Toque longo em slot vazio abre "novo agendamento" (Story 2.5).
