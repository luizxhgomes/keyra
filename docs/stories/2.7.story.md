# Story 2.7: Receita Prevista (exposição da view)

## Status

Ready

## Story

**Como** owner,
**Eu quero** ver a receita prevista dos próximos dias diretamente na agenda e no dashboard,
**Para que** eu tenha a previsão de caixa antes de o atendimento acontecer — principal diferencial vs ERPs genéricos.

## Complexidade

**T-shirt:** S (~2 pontos · 0,5 dia)

## Acceptance Criteria

### AC1 — View SQL

1. `v_receitas_previstas` já existe — confirmar colunas: org_id, data, receita_prevista, qtd_agendamentos.

### AC2 — UI Agenda

1. Card sticky no topo de `/agenda`: "Receita prevista hoje: R$ X" · "semana: R$ Y" · "mês: R$ Z".
2. Usa formatBRL; atualiza automaticamente quando um agendamento é criado/cancelado.

### AC3 — Dashboard

1. Na Story 4.4 o mesmo valor aparece no dashboard (esta story só deixa o cálculo pronto).

### AC4 — Revalidação

1. `getReceitaPrevista(range)` é uma **Server Action read-only** invocada a partir do Server Component da `/agenda`.
2. Atualização "automática" do card é provida por `revalidatePath('/agenda')` chamado dentro das mutations do agendamento (`createAppointment` na Story 2.5, `changeAppointmentStatus` na Story 2.6) — **não precisa polling client-side nem websockets neste MVP**.
3. Formatação dos valores com `formatBRL()` de `lib/money.ts` (Decimal.js + `pt-BR`).

## Tasks / Subtasks

- [ ] `app/(app)/agenda/actions.ts` → `getReceitaPrevista(range)` (Server Action read-only consultando `v_receitas_previstas`)
- [ ] `app/(app)/agenda/receita-card.tsx` (Server Component sticky no topo)

## Dependencies

- Story 2.4 (precisa do shell `/agenda` onde o card renderiza)
- View `v_receitas_previstas` ✅ (migration 015)
- Helper `formatBRL` em `lib/money.ts` ✅ (Story 1.1)
- Stories 2.5 / 2.6 indiretamente — quando elas chamam `revalidatePath('/agenda')`, este card também recompõe

## Definition of Done

1. Card sticky no topo de `/agenda` mostra os 3 valores (hoje · semana · mês) formatados em pt-BR.
2. Após criar um agendamento (2.5) ou cancelar (2.6), o valor recalcula sem reload manual.
3. Empty state quando não há agendamentos no range: card mostra "R$ 0,00" sem traços nem placeholder.
4. typecheck + lint + build verdes.
