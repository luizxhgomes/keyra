# Story 3.5: Registro Manual de Despesas

## Status

Ready

## Story

**Como** owner,
**Eu quero** registrar despesas (saídas) com classificação por categoria, fornecedor e data,
**Para que** o DRE reflita o resultado real do mês (FR-FI-02).

## Complexidade

**T-shirt:** M (~5 pontos · 1 dia)

## Contexto

`transactions` aceita `direction='debit'` para despesas. `expense_categories` tem `kind` (revenue/expense), seed inicial em migration 018. Despesas podem ser únicas ou recorrentes (caso de aluguel/energia mensal). MVP: apenas únicas; recorrência via "duplicar despesa do mês passado" no UI ou via Story 3.6 (custos fixos).

## Acceptance Criteria

### AC1 — CRUD `/financeiro/despesas`

1. Lista paginada (20/pág) com colunas: data, descrição, categoria, valor, fornecedor.
2. Filtro por período + categoria.
3. Form criação/edição: `gross_amount` (R$, obrigatório), `expense_category_id` (Select de kind=expense), `description` (texto), `reference_date` (data), `supplier_name` (opcional), `account_id` (default conta principal), `notes` (opcional).
4. `direction='debit'` setado pelo backend; `net_amount=gross_amount` (sem fee em despesas no MVP).

### AC2 — Soft-delete

1. Botão "Arquivar" não deleta — seta `deleted_at`. Despesas arquivadas somem da lista padrão.
2. Toggle "Mostrar arquivadas" para auditoria.

### AC3 — Validações

1. `gross_amount > 0` (Zod).
2. `reference_date` aceita passado/presente; futuro só com confirmação ("Lançar despesa futura?").
3. Categoria deve ser `kind='expense'` — Server Action filtra; UI já filtra no Select.

### AC4 — Integração com Plano de Contas (FR-FI-05)

1. Categorias prontas vindas do seed (limpeza de pele, peeling, botox, energia, aluguel, etc.) já aparecem no Select.
2. Botão "+ Nova categoria" abre dialog rápido (sem sair da página) e atualiza o Select.
