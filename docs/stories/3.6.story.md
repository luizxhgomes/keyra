# Story 3.6: Custos Fixos vs Variáveis

## Status

Done

## Story

**Como** owner,
**Eu quero** classificar despesas como fixas (mensais recorrentes) ou variáveis (eventuais),
**Para que** eu calcule rateio sobre a receita e entenda o custo de cada serviço (FR-FI-03, FR-CP-01/02).

## Complexidade

**T-shirt:** S (~3 pontos · 0,5 dia)

## Contexto

`expense_categories.kind` já distingue revenue/expense. Falta uma flag adicional (`is_fixed boolean`) que permite agrupar fixos mensais. Despesas variáveis vinculadas a insumos já entram via Story 2.3 (BOM); fixos são lançamentos manuais recorrentes.

## Acceptance Criteria

### AC1 — Migration 025

1. `ALTER TABLE expense_categories ADD COLUMN is_fixed boolean NOT NULL DEFAULT false`.
2. Reseed/UPDATE para classificar categorias do seed: aluguel/energia/internet/salário fixo = `is_fixed=true`; restante variável.

### AC2 — UI de gestão de categorias

1. Página `/financeiro/categorias` (sub-nav do financeiro) lista categorias agrupadas por kind (Receita/Despesa) com badge "Fixa" / "Variável".
2. Form criar/editar permite togglar `is_fixed` (apenas para kind='expense').

### AC3 — Visão consolidada `/financeiro/custos-fixos`

1. Página dedicada lista todas as transações `direction='debit'` cuja `expense_category.is_fixed=true` no período (default mês).
2. Mostra total mensal + média móvel (3 meses) para detectar variação.
3. Botão "Lançar todos os custos fixos do mês" duplica os fixos do mês anterior com `reference_date` ajustada (UX para não digitar 1 a 1).

### AC4 — Recorrência manual (não automática)

1. MVP não tem cron de despesas recorrentes — Luiz aciona manualmente. Recorrência automática vai para Phase 5+.
