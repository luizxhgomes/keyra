# Story 4.6: Indicadores no Dashboard

## Story

**Como** owner,
**Eu quero** ver indicadores-chave no dashboard (ticket médio, serviço mais vendido, taxa de comparecimento),
**Para que** eu identifique tendências da operação em uma piscada (FR-IN-01..03).

## Status

Done

## Complexidade

**T-shirt:** M (~5 pontos · 1 dia)

## Contexto

Indicadores são queries agregadas que ainda não têm view dedicada (decidido durante Phase 0 que cada indicador é simples o suficiente para virar query inline). Cada indicador é um card pequeno (mini-KPI).

## Acceptance Criteria

### AC1 — Card de indicadores no dashboard

1. **Ticket médio do mês**: `SUM(commands.total) / COUNT(commands paid)` no mês corrente.
2. **Serviço mais vendido (qty)**: `services.name` com mais `command_items.quantity` no mês.
3. **Serviço mais lucrativo**: `services.name` com maior soma de lucro (reusa lógica da Story 4.2).
4. **Taxa de comparecimento**: `(done) / (done + no_show + cancelled)` × 100, em %.

### AC2 — Layout

1. Grid 2x2 (4 cards pequenos).
2. Cada card: label · número absoluto · linha de comparativo "X a mais que mês passado" (sem %).

### AC3 — Período

1. Default: mês corrente. Filtro herda do dashboard.
