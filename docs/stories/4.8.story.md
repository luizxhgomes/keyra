# Story 4.8: Comparativo vs Meta Projetada

## Status

Ready

## Story

**Como** owner,
**Eu quero** definir uma meta mensal de receita/lucro e ver quanto faltou ou ultrapassei,
**Para que** eu mantenha a clínica direcionada a um objetivo (FR-IN-05).

## Complexidade

**T-shirt:** M (~5 pontos · 1 dia)

## Contexto

Tabela `goals` já existe no schema (migration 011) com campos `org_id`, `metric` (`revenue` | `profit` | `appointments`), `period_start`, `period_end`, `target_amount`. Esta story expõe CRUD + comparativo no dashboard.

## Acceptance Criteria

### AC1 — Página `/financeiro/metas`

1. Lista metas ativas e históricas (filtro por ano).
2. Form criar/editar meta: `metric` (Receita · Lucro · Agendamentos), `period_start` + `period_end` (default: mês corrente), `target_amount`.
3. Soft-delete via `deleted_at`.

### AC2 — Card "Meta do mês" no dashboard

1. Card visível se houver meta ativa para o mês corrente.
2. Mostra: `target_amount`, `current_value` (calculado conforme metric), `diferença absoluta` ("R$ X faltam para a meta" / "R$ X acima da meta"), `dias restantes no mês`.
3. Sem barra de progresso visual; usa apenas números absolutos + comparativo textual.

### AC3 — Cálculo de `current_value`

1. `metric='revenue'`: SUM(`transactions.net_amount`) WHERE direction=credit no período.
2. `metric='profit'`: receita − despesas no período (mesma lógica da DRE básica).
3. `metric='appointments'`: COUNT(`appointments`) WHERE status=`done` no período.

### AC4 — Sem meta cadastrada

1. Card no dashboard mostra: "Você ainda não definiu uma meta para [Mês]. [Criar meta](/financeiro/metas/nova)".

## Riscos

- **Meta retroativa**: usuário cria meta hoje cobrindo o mês passado — UX deve avisar mas permitir.
- **Múltiplas metas no mesmo período**: schema permite; UI default mostra a mais recente. Adicionar UNIQUE constraint só se confusão for documentada como real.

## Definition of Done

1. CRUD de metas funcionando.
2. Card no dashboard com diferença absoluta + dias restantes.
3. Empty state "criar meta" quando não há.
4. typecheck + lint + build verdes.
