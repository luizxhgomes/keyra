# Story h8.6: BetterStack uptime monitor + alerta WhatsApp

## Status

Draft

## Story

**Como** operador do KEYRA querendo saber primeiro que a clínica-piloto se uptime cair,
**Eu quero** BetterStack monitorando 3 endpoints críticos com alerta no meu WhatsApp,
**Para que** outage de produção me chegue em < 60 segundos e eu possa agir antes que a Camila perceba.

## Complexidade

**T-shirt:** XS (~1 ponto · 30 min)

## Acceptance Criteria

### AC1 — Conta BetterStack

1. Conta criada em [betterstack.com](https://betterstack.com) (free tier: 10 monitors, 3 min interval, alerta email + Slack/WhatsApp via Telegram).
2. Token salvo em `.keyra-secrets/betterstack-api.key` (chmod 600).

### AC2 — 3 monitores configurados

| Monitor | URL | Frequência | Critério verde |
|---------|-----|------------|----------------|
| Landing page | `https://usekeyra.com/` | 3 min | HTTP 200 + texto "KEYRA" no body |
| Login | `https://usekeyra.com/login` | 3 min | HTTP 200 + texto "Entrar" no body |
| Healthcheck | `https://usekeyra.com/api/healthcheck` | 1 min (cron mais agressivo) | HTTP 200 + JSON `status: 'ok'` |

1. Canal primário: **WhatsApp Business** do Luiz (via Telegram bot relay ou Twilio se gratuito não suportar WA direto).

3. Threshold: 2 falhas consecutivas dispara alerta (evita flapping).
4. Status page pública: `https://status.usekeyra.com` (BetterStack hospeda) — opcional, mas considerado.

### AC4 — Documentação

1. `docs/operations/uptime-monitoring.md` lista monitores, frequências, severity, runbook básico (o que fazer se cair).
2. Runbook: passo-a-passo "Vercel down? Supabase down? Sentry down?".

## Dev Notes

### Por que BetterStack vs Vercel checks

- Vercel built-in checks só monitoram domínios primários, não rotas específicas.
- BetterStack tem free tier suficiente, status page incluso, alerta multi-canal.
- Alternativa: Uptimerobot (também free) — escolha entre as duas pode ser tomada na hora; BetterStack é a recomendação por status page integrada.

### Frequência 1 min do healthcheck

Healthcheck dispara cron Vercel a cada 6h, mas BetterStack chama o endpoint a cada 1 min. O endpoint não precisa fazer query pesada nessa frequência — basta retornar status do último cron run. Otimização: cachear última latência em memória (Vercel function lifecycle), TTL 5 min.
