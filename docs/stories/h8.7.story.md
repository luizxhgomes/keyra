# Story h8.7: Performance budget enforced no CI

## Status

Draft

## Story

**Como** mantenedor do KEYRA preocupado com bundle creep ao longo do tempo,
**Eu quero** o CI bloquear merges que cresçam o bundle inicial além de 400KB gzipped,
**Para que** NFR-PE-01 (FCP < 1.5s 4G) continue cumprido sem ter que medir manualmente cada PR.

## Complexidade

**T-shirt:** S (~3 pontos · 0.5 dia)

## Acceptance Criteria

### AC1 — next-bundle-analyzer instalado

1. `@next/bundle-analyzer` instalado em `apps/web/package.json`.
2. `next.config.ts` envolvido com analyzer condicional via env `ANALYZE=true`.
3. Script `pnpm analyze` que builda com analyzer + abre dashboard HTML local.

### AC2 — Threshold no CI

1. Workflow `.github/workflows/perf-budget.yml` novo (ou estendido em `rls-tests.yml`).
2. Job `bundle-size`:
   - Roda `pnpm build`
   - Lê `apps/web/.next/build-manifest.json` + `app-build-manifest.json`
   - Calcula tamanho do bundle inicial do `/dashboard` (mais carregada) gzipped.
   - Falha se > 400KB.
3. Comment em PR (via `actions/github-script`) com tabela "Bundle delta vs main".

### AC3 — Lighthouse CI opcional

1. Vercel Analytics já mede Web Vitals em prod — verificar dashboard semanal manualmente é suficiente para MVP.
2. Story registra como follow-up: Lighthouse CI em PRs de páginas críticas (`/dashboard`, `/agenda`) como story futura `h9.x` se Vercel Analytics não der o sinal.

### AC4 — Documentação

1. `docs/operations/performance-budget.md` com:
   - Threshold atual (400KB gzipped initial JS no /dashboard)
   - Como rodar `pnpm analyze` local
   - O que fazer quando CI falha (lazy-load, dynamic import, tree-shake check)
2. README.md raiz ganha section "Performance" com link.

## Dev Notes

### Threshold 400KB

PRD NFR-PE-01 pede FCP < 1.5s em 4G. Pesquisa Vercel mostra que esse target requer ~200KB JS inicial parsed. Gzipped, target é ~400KB. Bundle atual é 325KB → margem de ~75KB para crescer.

### Por que /dashboard

É a página mais carregada — entra na primeira load + tem mais widgets/queries. Se ela passa, as outras passam.
