# Story i5.5: Stripe billing UI (trial 14d → subscription → Customer Portal)

## Status

Draft (bloqueada por D-PR1 — pricing absoluto)

## Story

**Como** dono do KEYRA pronto para comercializar,
**Eu quero** Stripe integrado com 3 planos (Start/Crescimento/Autoridade), trial de 14 dias sem cartão e Customer Portal,
**Para que** clínicas se cadastrem, escolham plano, paguem mensal/trimestral/anual e gerenciem cartão sem eu intervir.

## Complexidade

**T-shirt:** L+ (~10 pontos · 2.5 dias)

## Acceptance Criteria

### AC1 — Setup Stripe

1. Conta Stripe BR criada.
2. 3 produtos no Stripe: `keyra-start`, `keyra-crescimento`, `keyra-autoridade`.
3. Cada produto com 3 prices: mensal, trimestral (-10%), anual (-25%).
4. Trial de 14 dias configurado em todos os planos.
5. Stripe Tax habilitado para BR.
6. Webhook signing secret salvo em `STRIPE_WEBHOOK_SECRET`.

### AC2 — Schema mirror

1. Migration: `subscriptions (id, org_id UNIQUE, stripe_subscription_id, stripe_customer_id, plan, billing_cycle, status, trial_ends_at, current_period_end, cancel_at_period_end, ...)`.
2. RLS: org owner vê só sua subscription.
3. Source of truth: Stripe. DB é cache.

### AC3 — Onboarding atualizado

1. Após criar org, redireciona para `/billing/escolher-plano`.
2. Página com 3 cards (Start/Crescimento/Autoridade) + toggle mensal/anual.
3. Botão "Iniciar trial 14 dias" (sem cartão exigido) → cria subscription Stripe em `trialing`.
4. Trial 14d: full access ao plano escolhido.
5. Banner persistente "Trial expira em X dias — adicionar cartão" durante trial.

### AC4 — Webhook handler

1. `app/api/webhooks/stripe/route.ts` valida assinatura HMAC.
2. Trata eventos: `customer.subscription.created`, `updated`, `deleted`, `invoice.payment_failed`, `invoice.payment_succeeded`.
3. Atualiza `subscriptions` mirror + `organizations.plan`.
4. Idempotência via `stripe_event_id` table (evita reprocesso).

### AC5 — Customer Portal

1. Em `/configuracoes/cobranca`, botão "Gerenciar cobrança" → cria sessão Stripe Customer Portal e redireciona.
2. Cliente atualiza cartão, troca plano, cancela, baixa faturas — tudo no portal.
3. Cancelamento: define `cancel_at_period_end=true`; org vira readonly após `current_period_end`.

### AC6 — Feature gating

1. Middleware `lib/subscription/feature-gate.ts` checa plano da org antes de rotas Phase 5+:
   - Plano Start: NÃO pode acessar `/servicos/[id]/precificacao`, `/servicos/pacotes`, projeções, what-if, prontuário cliente.
   - Plano Crescimento: tudo da Start + Phase 5+6 sem integrações externas.
   - Plano Autoridade: tudo + Phase 7 (integrações).
2. Tentativa de acesso bloqueada → página `/billing/upgrade?feature={X}` com CTA "Migrar para Crescimento".

### AC7 — Inadimplência

1. `invoice.payment_failed` → email automático via Resend (template em pt-BR) + banner no dashboard "Pagamento pendente".
2. 7 dias após failure: suspensão soft (read-only).
3. 30 dias: cancelamento automático.
