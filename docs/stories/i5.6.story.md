# Story i5.6: MFA TOTP opt-in (D7 fechado)

## Status

Draft

## Story

**Como** dona de clínica preocupada com acesso indevido ao financeiro,
**Eu quero** habilitar MFA TOTP (Google Authenticator/Authy) na minha conta,
**Para que** mesmo se vazar minha senha de email, ninguém consiga ver minha DRE.

## Complexidade

**T-shirt:** S (~3 pontos · 0.5 dia)

## Acceptance Criteria

### AC1 — Configuração

1. Em `/configuracoes/seguranca`, card "Autenticação em duas etapas (2FA)".
2. Opt-in: gera QR code via Supabase Auth `mfa.enroll(factorType: 'totp')`.
3. User escaneia em app TOTP → digita código de 6 dígitos para confirmar.
4. Backup codes gerados (10 códigos de uso único) — download obrigatório.
5. Server Action `enableMFA()` com validação Zod.

### AC2 — Login

1. User com MFA habilitado: após magic link, pede código TOTP antes de redirecionar para dashboard.
2. Falha 5 vezes: bloqueia 15min.
3. Backup code aceito como fallback (consome 1 código).

### AC3 — Desabilitação

1. Botão "Remover 2FA" requer senha (mas KEYRA é passwordless… então requer **magic link novo + código TOTP atual**).
2. AlertDialog confirma: "Remover 2FA reduz segurança da sua conta. Continuar?"

### AC4 — Audit

1. `audit_log` registra `action='mfa_enabled'`, `'mfa_disabled'`, `'mfa_backup_code_used'`.

## Dependencies

- Supabase Auth nativa (já tem suporte TOTP).
- `h8.4` (LGPD) para audit trail.

## Definition of Done

1. Opt-in funciona end-to-end.
2. Login com MFA testado.
3. Backup codes funcionam.
4. Vitest cobre fluxo (mock Supabase Auth).
5. Audit_log captura eventos.
