# Story i5.7: Plano de contas customizável por org (D10)

## Status

Draft

## Story

**Como** Camila querendo categoria "Marketing Instagram" que não existe no plano padrão,
**Eu quero** adicionar/editar/desativar categorias do meu plano de contas,
**Para que** o DRE reflita a estrutura real da minha clínica, não só o template default.

## Complexidade

**T-shirt:** S (~3 pontos · 0.5 dia)

## Acceptance Criteria

### AC1 — CRUD de categorias

1. `/financeiro/categorias` (já existe — Story 3.6) ganha:
   - Botão "Nova categoria"
   - Edit em linha (nome, kind: revenue/fixed_cost/variable_cost, parent_id opcional)
   - Soft delete (`is_active=false`) — preserva histórico
2. Validação: não pode deletar/desativar categoria com transações ativas (avisa quantidade).
3. Hierarquia 1 nível (parent_id) — ex: "Marketing > Instagram", "Marketing > Google Ads".

### AC2 — Migração das categorias seed

1. Migration nova: `expense_categories.is_default boolean default false` para marcar as 29 do seed.
2. Categorias custom têm `is_default=false` + `created_by=user_id`.
3. UI distingue: defaults com label "padrão", custom com label "criada por você".

### AC3 — DRE atualizado

1. `/financeiro/dre` agrupa por hierarquia: parent → children somados.
2. Categorias inativas com transações antigas continuam aparecendo no histórico (mas não no select de novas despesas).

2. Hierarquia 1 nível funcionando.
3. Soft delete preserva histórico DRE.
